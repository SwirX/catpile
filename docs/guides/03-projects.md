## The Builder

```python
from catpile.builder import Builder

builder = Builder()

# Add compiled scripts
builder.add_script("loading.cat")
builder.add_script("game.cat")

# Link UI references (.catui resolves page. paths)
builder.link_ui("page.catui")

# Build final output
builder.build("output.json")
```

## UI Linker

The UI linker resolves `page.` path references to CatWeb global IDs.

### How It Works

1. Load `.catui` (generated by decompiler)
2. Build index: `page.Path.Name` ‚Üí `globalid`
3. Walk IR, find all `ObjectRef` nodes
4. Replace with resolved global IDs

```python
from catpile.ui import UILinker

linker = UILinker("page.catui")
resolved = linker.link(program)
print(f"Resolved {resolved} references")
```

### Path Format

```
Page.Folder.Element
Page.Folder.Subfolder.Button
Page.Script
```

The `.catui` paths map is a flat dict:

```json
{
  "paths": {
    "Page.FileLoader": "5s",
    "Page.FileLoader.Load": "@<",
    "Page.FileLoader.FileInput": "2\"",
    "Page.ThemeButton": "`f"
  }
}
```

## Page Metadata

Page-level properties are stored on the VFS root node during import and re-included:

- `favicon` - Favicon asset ID
- `title` - Browser tab title
- `background` - Page background color
- `description` - Meta description
- `thumbnail_id` - Share thumbnail

## Full Project Compilation

The editor's "Full Project" button:

1. Collects all scripts from VFS
2. Wraps each with `script "name":` directive
3. Sends to `POST /api/compile`
4. Receives compiled scripts
5. Builds UI tree from VFS
6. Merges scripts into correct positions
7. Returns complete page dict
